# 科技新闻深度分析模板

> 使用方法：告诉 HappyClaw "帮我分析一下这条科技新闻 [新闻标题/URL/事件描述]"
> HappyClaw 会自动按此模板启动多Agent协作流程

---

## 分析维度（8个）

### Agent 1: 事件还原 — 来龙去脉

**目标**：还原事件的完整时间线，让没关注过的人也能看懂发生了什么。

分析要点：
- **事件时间线**：关键节点按时间排列（起源→发酵→爆发→当前状态）
- **核心参与者**：谁做了什么？涉及哪些公司/团队/个人？
- **信息源梳理**：哪些渠道报道了这件事？官方说法 vs 民间声音？
- **事件性质**：是发布？事故？突破？争议？泄露？合作？竞争？
- **一句话概括**：用一句话说清这件事

### Agent 2: 技术解剖 — 这是什么科技 + 怎么做到的

**目标**：把技术讲透，让非技术背景的人也能理解核心原理。

分析要点：
- **技术分类**：属于哪个领域？（AI/芯片/生物科技/航天/新能源/量子计算/机器人等）
- **技术原理**：核心机制是什么？用类比解释（避免堆砌术语）
- **关键技术指标**：参数、性能数据、对比基准（和上一代/竞品比）
- **实现路径**：从0到1经历了哪些步骤？关键技术突破点在哪？
- **技术门槛**：为什么别人没做出来？难点在哪？

### Agent 3: 动因溯源 — 为什么会有这件事

**目标**：分析事件背后的驱动力，不只看表面原因。

分析要点：
- **直接原因**：触发这次事件的导火索是什么？
- **深层动机**：商业竞争？技术积累到了临界点？政策推动？市场需求爆发？
- **行业背景**：这件事放在行业大趋势里，处于什么位置？
- **为什么是现在**：同样的技术/想法，为什么在这个时间点爆发？
- **谁受益谁受损**：利益格局分析

### Agent 4: 问题与挑战 — 它在解决什么问题，又存在什么问题

**目标**：客观评价，既讲亮点也讲短板。

分析要点：
- **解决的核心痛点**：之前这个领域存在什么问题？这个科技怎么解决的？
- **技术局限性**：当前还不完美的地方？距离大规模商用还有多远？
- **已知风险**：安全隐患、伦理争议、监管空白、供应链风险？
- **被忽视的问题**：主流报道没提到但值得关注的隐患
- **争议焦点**：各方对这件事的主要分歧是什么？各自论据是什么？

### Agent 5: 影响评估 — 结果与连锁反应

**目标**：评估这件事的实际影响和潜在波及范围。

分析要点：
- **直接影响**：对相关公司/用户/行业的即时影响
- **产业链影响**：上下游供应链、合作伙伴、竞争对手会受到什么波及
- **用户层面**：普通人的生活会因此改变吗？怎么变？
- **资本/市场反应**：股价、投资趋势、融资动向
- **国际格局**：如果涉及中美博弈、技术封锁、标准争夺等
- **短期 vs 长期**：哪些影响是立竿见影的，哪些需要时间验证

### Agent 6: 行业生态 — 竞争格局与生态位

**目标**：把这件事放进行业地图里看，理解竞争态势。

分析要点：
- **主要玩家**：这个赛道上有哪些关键玩家？各自的优势和劣势
- **竞品对比**：同类技术/产品的横向对比（用表格直观呈现）
- **技术路线之争**：是否存在多条技术路线？各自前景如何？
- **生态成熟度**：开发者生态、社区活跃度、商业化程度
- **入局门槛**：新玩家想进入这个领域的难度和路径

### Agent 7: 延伸思考 — 启示与趋势预判

**目标**：从单一事件提炼出有价值的洞察和趋势判断。

分析要点：
- **历史类比**：有没有类似的历史事件？当时的发展轨迹是怎样的？
- **给从业者的启示**：技术人员/创业者/投资人能从中学到什么？
- **趋势预判**：基于这件事，未来6-12个月这个方向会怎么走？
- **值得关注的时间节点**：接下来有哪些关键事件/发布节点值得跟踪
- **思维拓展**：这件事背后的逻辑能不能迁移到其他领域？

### Agent 8: 事实核查 — 信息的可信度审计

**目标**：过滤噪音，标注信息的可靠程度。

分析要点：
- **官方信息 vs 媒体渲染**：哪些是确认的事实，哪些是媒体猜测
- **数据验证**：报道中提到的数据是否可信？来源是否权威
- **常见误区**：关于这件事，大众最容易误解的地方
- **信息缺口**：哪些关键信息目前还不知道？为什么？
- **可信度标注**：对每个核心论断标注可信度等级

---

## 并行执行策略

分两批并行：

**第一批（4个Agent同时）**：
- Agent 1 事件还原
- Agent 2 技术解剖
- Agent 3 动因溯源
- Agent 8 事实核查

**第二批（4个Agent同时，依赖第一批结果）**：
- Agent 4 问题与挑战
- Agent 5 影响评估
- Agent 6 行业生态
- Agent 7 延伸思考

> 第二批 Agent 可引用第一批的输出，避免重复调查。

## 最终报告结构

```markdown
# [新闻标题] — 深度分析

> 分析时间 | 信息源数量 | 可信度总评

## 一句话总结
（50字以内）

## 事件时间线
（可视化的时间轴）

## 技术解读
（原理、指标、实现路径）

## 为什么是现在
（直接原因 + 深层动机 + 行业背景）

## 竞争格局
（横向对比表格）

## 影响与连锁反应
（分短期/中期/长期）

## 问题与争议
（已知局限、风险、各方分歧）

## 趋势与启示
（历史类比、趋势预判、行动建议）

## 信息可信度说明
（各核心论断的可信度标注）

## 附录：关键信息源
（引用链接列表）
```

## 注意事项

1. **新闻有时效性**：优先抓取最新信息，标注信息截止时间
2. **多方信源交叉验证**：不依赖单一媒体，至少对比 2-3 个来源
3. **区分事实与观点**：明确标注哪些是确认事实、哪些是分析推断
4. **避免技术傲慢**：用类比讲原理，让非技术人员也能看懂
5. **保持中立**：不站队，呈现多方观点
6. **成本控制**：每批最多 4 个 SubAgent，总计 8 个

# Multi-Agent 深度分析模板

> 使用方法：告诉 HappyClaw "帮我深度分析一下 [主题/URL/GitHub仓库]"
> HappyClaw 会自动按此模板启动多Agent协作流程

---

## Phase 1: 主题分解（主Agent）

根据分析目标，自动拆分为 5-8 个分析维度。常见维度模板：

### 软件项目分析
1. **项目概览**：项目定位、目标用户、解决的问题、当前状态
2. **核心架构**：技术栈、系统架构、模块划分、数据流
3. **关键实现**：核心算法/机制、设计模式、技术创新点
4. **依赖生态**：第三方依赖、版本策略、兼容性
5. **代码质量**：代码结构、测试覆盖、文档完整度
6. **性能与安全**：性能瓶颈、安全风险、已知漏洞
7. **战略分析**：竞品对比、市场定位、发展潜力
8. **实践评估**：上手难度、适用场景、推荐指数

### 技术趋势分析
1. **技术背景**：历史演进、动机、解决什么问题
2. **核心原理**：技术原理、关键论文/实验
3. **实现方案**：主流框架、工具链、最佳实践
4. **生态现状**：社区活跃度、商业支持、标准化进展
5. **对比分析**：与替代方案的优劣对比
6. **落地建议**：适用场景、风险提示、推荐路径

---

## Phase 2: 并行分析（SubAgent 并行）

每个维度启动一个 `general-purpose` SubAgent：

```
Agent 1 → 维度1分析（独立上下文）
Agent 2 → 维度2分析（独立上下文）
Agent 3 → 维度3分析（独立上下文）
...
Agent N → 维度N分析（独立上下文）
```

每个 SubAgent 的 Prompt 结构：
```
你是深度分析团队的 [维度名称] 专家。

分析主题：[主题]
分析维度：[具体维度描述]
上下文：[主题的背景信息、URL、仓库地址等]

要求：
1. 深入分析，给出具体细节而非泛泛而谈
2. 如有数据/指标，给出具体数字
3. 标注信息的可信度（确定/推测/需要进一步验证）
4. 如果发现了值得深入的方向，在末尾列出 "待追问" 项

输出格式：Markdown，带清晰的层级标题
```

**并行约束**：
- 最多同时启动 5 个 SubAgent（避免资源争抢）
- 超过 5 个维度时分批执行
- 每个 SubAgent 的上下文相互隔离

---

## Phase 3: 收集与清洗（主Agent）

所有 SubAgent 返回后，主Agent执行：

1. **去重**：移除多个Agent重复覆盖的内容
2. **标记矛盾**：如果Agent之间结论冲突，标记待验证
3. **补充追问**：针对各Agent的 "待追问" 项，可启动第二轮专项分析
4. **优先级排序**：按信息价值对内容排序

---

## Phase 4: 顺序合成（主Agent）

单个Agent一口气合成最终报告，保证连贯性：

```
报告结构：
├── 摘要（200字以内）
├── 分析正文（按Phase 1的维度组织）
│   ├── 维度1（来自Agent 1的成果，经清洗后）
│   ├── 维度2（来自Agent 2的成果，经清洗后）
│   └── ...
├── 交叉发现（跨维度的关联洞察）
├── 风险与局限（分析过程中的不确定项）
└── 结论与建议
```

**合成规则**：
- 统一术语和格式
- 添加跨维度的交叉引用
- 在不确定性高的地方标注 [待验证]
- 结论要有数据支撑，避免空泛建议

---

## Phase 5: 质量检查

最终报告交付前的检查清单：

- [ ] 各维度是否都有实质性内容（非泛泛而谈）
- [ ] 是否有矛盾结论未标注
- [ ] 数据/引用是否标注来源
- [ ] 报告结构是否符合模板
- [ ] 结论是否可操作

---

## 注意事项

1. **不做摘要传递**：Agent之间的信息传递保留原始内容，不压缩
2. **最终报告顺序写**：不用并行合成，保证文风一致
3. **人工兜底**：不确定的内容标注 [待验证]，不瞎编
4. **成本控制**：每次分析的 SubAgent 总数控制在 5-8 个
